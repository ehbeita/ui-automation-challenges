Actions

Using an UI automation framework, resolve the following challenges of actions being performed on DOM elements

Exercise 1

Typing and sending keys:
  • Type fake@email.com inside the email input. Then validate that: 
    Assert that the value of the input equals to "fake@email.com"
  • Fill the email input with fake@email.com and send the keyboard keys: CTRL + A (Win) or COMMAND + A (Mac) to select all and then DELETE. Validate that: 
    Assert that the input value is empty after the delete key is sent
  • Given the disabled text area, find a way to force typing the words Text typed inside disabled textarea into it. Validate that: 
    Assert that the text area contains "Text typed inside disabled textarea"

Exercise 2

Part 1: Focus
  • Given the password input, set the cursor focus on it and validate that: 
    The label should turn into orange color
    And the input should show an orange border
Part 2: Blur
  • Given the Full Name input, focust on it and then loose that focus (blur). Then validate that: 
    The label should turn into red color
    And the input should show a red border

Exercise 3

Submitting a form:
  • Given the coupon code field, fill it with a string and then submit the form. After submitting the form validate that: 
    The following text should be displayed: "Your form has been submitted!"
    Note: You can send the enter key to trigger the form submit

Exercise 4

Part 1: Popover and canvas
  • Click the red button with the text "Click to toggle popover", then validate that: 
    Make sure that a popover element shows up
  • Interact with the canvas element to draw red dots by clicking on the following coordinates: 
    (80, 75), (170, 75), (80, 165), (100, 185), (125, 190), (150, 185), (170, 165)
    Then validate that: 
    The red dots should show up in their corresponding position
    You can use a visual validation tool like percy or Applitools
Part 2: Multiple clicks and force
  • Find the elements with the text "click me", "and me", "and me"... locate them using one single selector. Once you find them, iterate the list, click each of them. Then validate that: 
    A popover is shown for each clicked element
  • The button "I'm being covered" has an element on top of it. Write an automated script that can force click on that element, even if it is being covered by another element.

Exercise 7

Part 1: Checkbox
  • Check all checkboxes that are not disabled. Validate that: 
    Checkboxes should be checked, except the disabled one
  • Check the disabled checkbox. Validate that: 
    Only the disabled checkbox should be checked
  • Check the checkbox that contains text "Example checkbox". Validate that: 
    The checkbox with text "Example checkbox" should be checked
Part 2: Radio
  • Check the radio that contains text "radio1". Validate that: 
    Radio with text "radio1" should be checked
  • Check the radio that contains text "radio2". Validate that: 
    Radio with text "radio2" should be checked
  • Check the disabled radio. Validate that: 
    The disabled radio should be checked

  • Uncheck the disabled checkbox. Validate that:

Exercise 10

Part 3: Both scroll
